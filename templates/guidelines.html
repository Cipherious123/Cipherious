Guidelines to Create a Combination

Follow these steps and tips to create combinations effectively:

  • Using Integer Base changer: Password should be of the format: base of the plaintext during encryption, base to convert it to, system of input number, system to convert it to
  • Using Integer Base changer: We represent normal system as 'n', alpha as 'a', and unicode as 'u'
  • Using Integer Base changer: For example: '10 40 n u' means we convert input (which is in base 10 and normal system) to base 40 and unicode system
  • Password Requirements: Max length: 100 characters. Combo name: Max length: 25 characters. Use basic Latin Unicode only.
  • Substitution & Caesar: Best for smaller texts. Larger texts may show patterns more easily.
  • Scrambler: Best for larger texts as it jumbles letters, making patterns harder to detect.
  • Strong combo: Always use polyalphabetic ciphers, such as Vigenere, Integer Base Changer, and especially AES, for a stong combination
  • Random Passwords: Leave any password blank to auto-generate it.
  • Save Progress: Always save the combination name before adding steps. Complete the combination after adding all steps.
